# Retail / POS Support Framework

Retail and POS systems are business-critical because they can affect sales, payments, stock lookup, returns, clienteling, and customer experience.

## Troubleshooting Flow

Use the following flow:

1. Scope
2. Impact
3. Device
4. Network
5. Account / Access
6. Application
7. Payment / Peripheral
8. Escalation

## 1. Scope

Ask:

- Is this affecting one user, one device, the whole store, or multiple stores?
- Are other staff able to log in?
- Are other devices working?
- Are other stores reporting the same issue?

## 2. Impact

Ask:

- Is the store still able to trade?
- Are payments affected?
- Are sales, returns, inventory lookup, or clienteling affected?
- Is there a workaround?

## 3. Device

Check:

- Is the POS terminal, iPad, or device powered on?
- Has the app or device been restarted?
- Is the device frozen or slow?
- Is the correct app being used?
- Is the date and time correct?

## 4. Network

Check:

- Is Wi-Fi or ethernet connected?
- Can other devices access the internet?
- Are other cloud services working?
- Is VPN required?

## 5. Account / Access

Check:

- Can the user log in?
- Is the issue affecting one staff member or all staff?
- Has the user’s password changed?
- Does the user have the correct role or permissions?

## 6. Application

Check:

- What error message appears?
- Did the issue start after an update?
- Is the issue with all functions or one function only?
- Is there a known outage?

## 7. Payment / Peripheral

Check:

- Is the payment terminal powered on?
- Is the receipt printer online?
- Is the barcode scanner connected?
- Are all card payments failing or only one transaction?
- Is there an error code?

## 8. Escalation

Escalate quickly if:
- Store trading is impacted
- Multiple users or devices are affected
- Payment processing is unavailable
- The issue appears vendor-specific
- A business-critical system is unavailable

## Escalation Notes Should Include

- Store/location
- User affected
- Device affected
- Business impact
- Error message
- Screenshots if available
- Troubleshooting already completed
- Workaround status

## Interview Summary

For retail system issues, I would prioritise based on store impact and customer experience. I would isolate whether the issue is device, network, account, application, payment, peripheral, or vendor-related, then escalate quickly with clear documentation if trading is affected.

# Ticket Prioritisation Framework

## Priority Factors

When prioritising IT support tickets, consider:

1. Business impact
2. Urgency
3. Number of users affected
4. Location affected
5. Whether a workaround exists
6. Whether the issue affects trading or customer experience
7. Whether the issue involves a business-critical system
8. Security or access risk

## Example Priority Levels

### High Priority

- Entire store cannot access POS system
- Payment terminal outage affecting sales
- Multiple users unable to access Microsoft 365
- Executive or VIP user unable to work with no workaround
- Security or access issue involving sensitive systems

### Medium Priority

- Single user cannot access Outlook desktop but Outlook Web works
- One staff member cannot access a business app but another user can assist
- Printer issue with alternative printer available
- OneDrive sync issue with manual workaround

### Low Priority

- Software installation request with no urgent deadline
- General how-to question
- Non-urgent hardware accessory request
- Documentation update

## Interview Summary

I would prioritise tickets based on business impact, urgency, number of users affected, whether a workaround exists, and whether the issue affects store trading or customer experience.
